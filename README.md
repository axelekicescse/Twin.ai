# Twin.ai - Authentic Personality Replication Platform

A Next.js web application that allows fans to interact with AI twins of real creators. The core feature is an **Authentic Personality Replication Engine** that faithfully replicates a creator's unique personality traits, communication style, and way of thinking.

## ⚠️ Important Disclaimer

This application is a **fan-made AI demo**. The AI responses are generated by OpenAI and may be inaccurate. The app explicitly states that it is an AI simulation and does not claim to be the real person.

## Features

- **Authentic Personality Replication**: Core engine that faithfully replicates creator personalities
- **Chat Interface**: Stream responses token-by-token for a natural conversation experience
- **iPhone Messages UI**: Beautiful chat interface mimicking iOS Messages with avatars and labels
- **Multiple Personas**: Chat with different creator personalities
- **Streaming Responses**: Real-time streaming of AI responses using Server-Sent Events (SSE)

## Setup Instructions

### Prerequisites

- Node.js 18+ installed
- OpenAI API key

### Installation

1. Clone or download this repository

2. Install dependencies:
```bash
npm install
```

3. Create a `.env.local` file in the root directory:
```bash
OPENAI_API_KEY=your_openai_api_key_here
```

4. Run the development server:
```bash
npm run dev
```

5. Open [http://localhost:3000](http://localhost:3000) in your browser

## How It Works

### The Functional Proof Loop

To prove that Twin.ai works, the following loop must function:

1. A fan sends a message
2. The Personality Replication Engine analyzes the message
3. The system generates a response aligned with the creator's real personality
4. The output feels authentic and consistent

**If this loop works, Twin.ai works.**
**If this loop fails, Twin.ai does not exist.**

### Personality Replication System

The system uses a sophisticated prompt engineering approach:

1. **Base Disclaimer**: Clear AI simulation notice
2. **Safety Guardrails**: Boundaries and constraints
3. **Personality Replication Core**: Instructions for authentic replication
4. **Conversation Authenticity**: Rules for natural conversation
5. **Persona Details**: Bio, style rules, principles, boundaries, examples
6. **Response Style**: Guidelines for authentic communication

Each persona includes:
- **Bio**: Background information about the creator
- **Style Rules**: How the creator typically communicates
- **Principles**: Core values and beliefs
- **Boundaries**: What the creator would NOT do or say
- **Examples**: Real examples of the creator's communication style

## API Routes

### `POST /api/chat`

Accepts a chat request with messages and returns a streaming response.

**Request Body:**
```json
{
  "messages": [
    { "role": "user", "content": "Hello" }
  ],
  "personaId": "naval"
}
```

**Response:** Server-Sent Events stream with `data: {"content": "..."}` chunks
